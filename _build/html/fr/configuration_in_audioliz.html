Configuration dans Audioliz¶

1. Créer une question¶

Vous pouvez modifier/créer autant de questions que vous le souhaitez dans votre feuille de suivi. Il existe trois types de questions :

1.1 Question textuelle¶

Vous pouvez choisir une question textuelle lorsque la réponse est un paragraphe. Par exemple : « Donnez-moi le résumé de l’appel » ou « Quels sujets ont été abordés lors de l’appel ? »

Si vous avez une question oui/non, il est préférable de ne pas sélectionner ce type de question.

3. Gestion des métadonnées dans Audioliz¶

3.1 Ajout ou modification des métadonnées¶

Les métadonnées désignent les informations contextuelles supplémentaires rattachées à un appel (par exemple, le nom de la campagne, l’ID CRM ou le canal). Certaines métadonnées sont stockées automatiquement dans le champ crm_metadata de la base de données, mais vous pouvez également définir et gérer des champs de métadonnées personnalisés pour des actions spécifiques.

Pour gérer les métadonnées d’une action spécifique :

  • Accédez à la page Actions

  • Sélectionnez la fiche d’évaluation à configurer en cliquant sur l’icône œil

  • Ouvrez l’onglet Métadonnées

Vous pouvez :

  • ➕ Ajouter une nouvelle métadonnée en cliquant sur l’icône plus (+)

  • ✏️ Modifier une métadonnée existante en cliquant directement sur la ligne concernée

Pour chaque champ de métadonnée, vous pouvez définir :

  • Name : le nom interne de la métadonnée, affiché dans la page Actions.

  • Label : le nom affiché dans la page d’appel.

  • Group : la section de la page d’appel où cette métadonnée apparaîtra.

  • Default value : la valeur par défaut utilisée si la métadonnée est manquante ou vide dans les données de l’appel.

Tip : si vous souhaitez que la question soit posée (c’est-à-dire incluse dans l’invite envoyée à l’IA), même lorsque la métadonnée est présente mais vide (c’est-à-dire définie pour l’appel sans valeur), définissez Default value sur un espace unique (`` ``).

Cela garantit que l’espace réservé à la métadonnée sera remplacé par une chaîne vide, et que la question ne sera pas ignorée lors de l’analyse.

3.2 Pourquoi les métadonnées sont importantes dans les questions¶

Certaines données nécessaires à une question (comme le nom de la campagne ou le type de produit) varient d’un appel à l’autre. Plutôt que de créer plusieurs versions de la même question, vous pouvez insérer dynamiquement des métadonnées en utilisant le symbole $ dans le texte de la question.

4. Gestion des rôles utilisateurs dans Audioliz¶

Dans Audioliz, chaque utilisateur possède un rôle définissant ce qu’il peut faire et accéder sur la plateforme. Les rôles servent à contrôler les autorisations en fonction des responsabilités : lecture des données, modification des fiches d’évaluation, gestion des utilisateurs ou commentaires sur les appels.

4.1 Rôles prédéfinis dans Audioliz¶

Rôle Description Admin : Accès complet à toutes les fonctionnalités Superviseur : Peut tout faire sauf l’administration des utilisateurs Superviseur externe : Comme les agents, mais peut aussi commenter les appels et modifier les métadonnées Agent : Peut consulter les appels et répondre aux grilles, mais ne peut pas gérer les utilisateurs ni les paramètres

Vous pouvez consulter cette liste en naviguant vers Configuration > Utilisateurs > RÔLES :

Cliquer sur un rôle (par exemple admin) affichera les permissions détaillées associées :

Les permissions contrôlent quels rôles peuvent accéder ou effectuer certaines opérations sur les appels

Étapes : accédez à Configuration > Utilisateurs dans le menu de gauche.

Vous verrez la liste des utilisateurs. Cliquez sur l’icône œil pour afficher les profils :

🔎 Astuce : si vous ne voyez pas les options de rôle ou ne pouvez pas les modifier, cela signifie que votre compte n’a pas les privilèges admin. Vous devrez contacter un administrateur pour mettre à jour les rôles.

5. Signification des champs Date sur la page d’appel¶

  • Date : date d’initiation de l’appel ou de l’email envoyé ; elle correspond au jour où cette action a été enregistrée dans le CRM du client.

  • Time : heure exacte à laquelle l’appel ou l’email a été créé dans le CRM du client, en complément du champ Date.

  • Creation : date à laquelle l’appel ou l’email est reçu dans Audioliz ; en d’autres termes, la date de création de cet élément sur notre plateforme.

  • Last update : date à laquelle cet enregistrement d’appel ou d’email a été modifié pour la dernière fois sur la page d’appel.
